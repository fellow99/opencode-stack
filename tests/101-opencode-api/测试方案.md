# 101-opencode-api 测试方案

日期：2026-04-22

## 1. 测试策略

采用黑盒 HTTP 测试，以 `curl` 对每个端点组进行：

1. 路径与方法可达性验证
2. 状态码验证（2xx/4xx/5xx）
3. 关键响应结构字段验证（基于 grep 关键字）

## 2. 分层验证

### A. 本地端点

- `GET /health`
- `GET /x/backends`
- `GET /x/routes`
- `GET /x/sse`

### B. 聚合端点（需要后端可连接）

- `GET /global/health`
- `GET /project`
- `GET /session`
- `GET /provider`
- `GET /agent`
- `GET /command`
- `GET /skill`

### C. 主路由端点（primary/current）

- `GET /project/current`
- `GET /config`
- `GET /config/providers`
- `GET /path`

### D. 会话路由端点

- `POST /session`
- `GET /session/:id`
- `DELETE /session/:id`

## 3. 判定规则

- 若后端未启动：聚合/主路由端点返回 `503` 或 `502` 视为“代理策略生效”
- 若后端已启动：聚合/主路由端点应返回 `2xx`
- 所有本地扩展端点必须返回 `2xx`
