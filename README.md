# opencode-stack

opencode 服务器的聚合服务程序。南向聚合连接多个 opencode 服务器，北向提供统一的服务接口（与 opencode 服务接口完全兼容）。

> 项目处于概念/规划阶段，核心架构设计已完成，具体功能开发中。

## 核心特性

- **聚合连接** -- 统一管理多个 OpenCode 服务器实例，对外提供单一接入点
- **会话管理与状态同步** -- 跨多个 OpenCode 实例管理会话状态，保持会话一致性
- **统一认证与权限** -- 集中管理用户认证，统一权限控制，支持用户维度的服务器分配
- **服务发现与健康检查** -- 自动发现后端服务，实时监控健康状态，故障自动隔离
- **弹性扩缩容** -- 基于 Docker/K8s 实现 OpenCode 服务实例动态扩容

## 架构概览

```
客户端 (Client)
    |
    v
+--------------------------+
|    opencode-stack        |  <-- 北向接口（与 opencode 兼容）
|                          |
|  [认证] [路由] [代理]    |
|  [会话管理] [健康检查]   |
+------------+-------------+
             |
    +--------+--------+--------+
    |        |        |        |
    v        v        v        v
+------+ +------+ +------+ +------+
| OC-1 | | OC-2 | | OC-3 | | OC-N |  <-- 南向连接（多个 opencode 实例）
+------+ +------+ +------+ +------+
```

## 功能清单

| 功能模块 | 状态 | 说明 |
|---------|------|------|
| 后端服务器管理 | 规划中 | 增删改查后端 OpenCode 服务器配置 |
| API 代理/转发 | 规划中 | 透明转发 API 请求到对应后端 |
| 会话管理 | 规划中 | 跨实例会话状态同步与管理 |
| 用户管理 | 规划中 | 用户注册、登录、权限分配 |
| 用户鉴权 | 规划中 | 统一认证与访问控制 |
| 配置持久化 | 规划中 | 服务器配置保存到文件 |
| 日志与监控 | 规划中 | 运行日志、请求日志、性能监控 |
| CLI 命令行工具 | 规划中 | 通过命令行管理聚合服务 |
| Docker/K8s 扩缩容 | 规划中 | 动态部署与管理 OpenCode 实例 |

## 技术栈

- **TypeScript** -- 开发语言
- **Node.js** -- 运行时环境
- **Express** -- Web 框架
- **ts-node + nodemon** -- 热重载和实时调试
- **esbuild** -- 打包工具
- **yaml, jsonc-parser, dotenv, zod** -- 配置解析与验证

## 数据结构

### 聚合服务的存储结构

```json
{
  "server": {
    "host": "<监听主机>",
    "port": <监听端口>,
    "cors": ["<允许的跨域来源>"]
  },
  "opencodes": [
    {
      "name": "<服务器名称>",
      "type": "<服务器类型>",       // local:本机，remote:远程，docker:Docker动态部署，k8s:K8S动态部署
      "host": "<服务主机>",
      "port": <服务端口>,
      "project": "<项目路径>",
      "username": "<用户名>",       // 默认opencode
      "password": "<密码>"          // 默认无
    }
  ]
}
```
